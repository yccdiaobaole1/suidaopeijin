## 一、公式核查结果（已验证正确）

### ✅ 1. 小偏心受压钢筋应力公式 (calculator.py:288)

**之前错误结论**: 称分母应为 `(ξb - 1)` 而非 `(ξb - 0.8)`

**规范原文** (Q/CR 9129-2018, 9.2.6-5, p.73):
```
σs = fy · (x/h0i - 0.8) / (ξb - 0.8)
```

**代码实现**:
```python
sigma_s = fy * ((x_trial / h0 - 0.8) / (xi_b - 0.8))
```

**核查结论**: ✅ **公式完全正确**。之前审查混淆了GB 50010的公式，GB 50010使用 `(ξb - 1)`，但Q/CR 9129-2018明确使用 `(ξb - 0.8)`。

MIN_RHO_TOTAL = 0.004   # 0.4%

### ✅ 3. 抗剪公式无βc系数 (calculator.py:122-132)

### ✅ 4. 斜截面受剪承载力Vs项除以γd (calculator.py:96-120)

## 二、真实存在的问题

### 🔴 P0 - 致命级问题（必须立即修复）

#### 问题1: NSGA-II主循环归一化值混淆

**文件**: `src/nsga2_solver.py`
**位置**: 第115-117行
**严重程度**: 致命

**问题描述**:
```python
# 第115-117行
combined = population + offspring
combined = self._fast_non_dominated_sort(combined)
combined = self._normalize_objectives(combined)
```

`population` 中个体的 `objs` 是上一代归一化后的 [0,1] 值，而 `offspring` 的 `objs` 是本代原始值。二者混合后再归一化，导致非支配排序在混乱的目标空间上进行，前沿划分错误。

**影响**: 优化结果不可信，帕累托前沿可能包含被支配的解。

**修复方案**:
```python
# 合并前先重置为原始值
for ind in population:
    ind.objs = ind.raw_objs.copy()
combined = population + offspring
combined = self._fast_non_dominated_sort(combined)
combined = self._normalize_objectives(combined)
```

**参考**: `exhaustive_enumerate` 方法中已正确实现了重置逻辑（第206行）。

### 🟠 P1 - 严重级问题（尽快修复）

#### 问题2: 纯弯构件逻辑缺失

**文件**: `src/calculator.py`
**位置**: 第250-251行
**严重程度**: 严重

**问题描述**:
当轴力 N≈0 时，代码走偏心受压路径，e0→∞，safety_factor→∞，纯弯构件被错误判定为绝对安全。

**规范依据**: Q/CR 9129-2018, 9.2.4 (p.71) - 受弯构件应按专门公式验算：
```
M ≤ (1/γd)[fc·b·x·(h0 - x/2) + fy'·As'·(h0 - as')]
```

**影响**: 对于明洞衬砌，拱形结构所有截面都有轴力，N≈0情况极少，实际影响有限。

**修复方案**:
```python
# 在 verify_structure 开头增加判断
if abs(N_d) < 1e-3:  # 纯弯构件
    # 按9.2.4公式验算
    x = (fy * As - fy_prime * As_prime) / (fc * b)
    Mu = (fc * b * x * (h0 - x/2) + fy_prime * As_prime * (h0 - as_prime)) / 1.0
    safety_factor = Mu / abs(M_d) if abs(M_d) > 1e-6 else 999.0
    return {...}
```

#### 问题3: 受拉构件未区分

**文件**: `src/calculator.py`
**位置**: 第250行
**严重程度**: 严重

**问题描述**:
```python
N_d_pos = abs(N_d)  # 将拉力当压力处理
```

轴力为负（受拉）时，应按偏心受拉公式计算，而非偏心受压。

**影响**: 明洞衬砌仰拱某些截面可能出现拉力，当前代码会给出错误的承载力判断。

**修复方案**:
```python
if N_d < -1e-3:  # 受拉构件
    # 按偏心受拉公式验算（需参照GB 50010或增加专门函数）
    return self._verify_eccentric_tension(...)
elif N_d > 1e-3:  # 受压构件
    # 现有偏心受压逻辑
else:  # 纯弯
    # 问题2的处理
```

#### 问题6: β₁硬编码不支持高强混凝土

**文件**: `src/calculator.py`
**位置**: 第272行
**严重程度**: 中等

**问题描述**:
```python
beta_1 = 0.8  # 硬编码
```

规范要求C50以上混凝土β₁应取更小值（C80时β₁=0.74）。

**影响**: 本项目明洞衬砌通常用C30~C50，实际影响不大。

**修复方案**:
```python
def _get_beta1(fc_value):
    """根据混凝土强度等级确定β₁"""
    if fc_value <= 50:
        return 0.8
    elif fc_value <= 80:
        return 0.8 - (fc_value - 50) * 0.002  # 线性插值
    else:
        return 0.74
```
