# 明洞衬砌配筋流程总结

## 概述

明洞衬砌配筋采用三阶段设计流程：

1. **主筋配置**：NSGA-II多目标优化算法
2. **拉筋配置**：启发式搜索 + 帕累托前沿筛选
3. **分布筋配置**：规则式配置

---

## 一、主筋配置流程（NSGA-II多目标优化）

**代码位置**：`src/nsga2_solver.py`

### 1.1 初始化阶段

**步骤1：启发式边界估算**（第46-58行）

- 调用 `_inverse_solve_min_area` 估算基准配筋面积 `a_base`
- 定义有效搜索区间：`[0.8×a_base, 1.5×a_base]`
- 计算最大允许间距：`max_s = get_max_spacing_main(h)`

**流程说明**：

  1. 判断截面高度：
    - 如果 h ≤ 150mm：返回 200mm
    - 如果 h > 150mm：返回 min(1.5×h, 250mm)
  2. 计算规则：
    - 小截面（h≤150mm）：固定最大间距200mm
    - 大截面（h>150mm）：按1.5倍截面高度，但不超过250mm

  **示例**：

  | 截面高度 h | 计算过程 | 最大间距 |
  |-----------|---------|
  | 100mm | h≤150 | 200mm |
  | 150mm | h≤150 | 200mm |
  | 200mm | min(1.5×200, 250) = min(300, 250) | 250mm |
  | 300mm | min(1.5×300, 250) = min(450, 250) | 250mm |
  | 500mm | min(1.5×500, 250) = min(750, 250) | 250mm |

  结论：主筋最大间距上限为250mm，对于大截面按1.5倍高度计算，小截面固定200mm。

**步骤2：种群初始化**（第61行）

- 每个个体基因：`[grade_idx, diameter_idx, spacing_idx]`
- 搜索空间：2等级 × 9直径 × 16间距 = 288种组合

对每个个体执行以下步骤：

**步骤1：解码基因**

## 二、拉筋配置流程（启发式搜索 + 帕累托筛选）

**代码位置**：`src/structural_solver.py` 的 `design_stirrups` 方法

### 2.1 计算所需配箍率（第427-455行）

**步骤1：遍历荷载工况**

对每个荷载工况计算：
- 调用 `_calc_shear_without_stirrup` 计算无拉筋抗剪能力
- 剪力缺口：`deficit = max(0, V_d - Vc)`
- 强度需求：`required_strength = deficit / (fyv × h0)`
- 截面限制比：`limit_ratio = V_d / V_max`

**拉筋截面限制比详解**
                                                    3s)
  1. 基本概念
                                                                                                                                                                             截面限制比：limit_ratio = V_d / V_max
                                                                                                                                                                             - V_d：设计剪力（荷载产生的剪力需求）                     t
  - V_max：截面抗剪承载力上限（混凝土斜压破坏控制）

  2. 物理意义

  V_max 的作用：
  - 防止混凝土腹部斜压破坏
  - 即使配置再多拉筋，剪力也不能超过V_max
  - 这是截面尺寸的固有限制

  limit_ratio 的含义：
  - < 1.0：截面安全，未达到斜压破坏上限
  - ≥ 1.0：截面超限，需增大截面尺寸
  - 越小越好，说明截面储备充足

  3. 计算公式

  代码位置：src/calculator.py 第119-129行

  规范依据：Q/CR 9129-2018 第9.2.7条

   **1. 计算系数（根据高宽比）**
  ratio_hw_b = h_w / b

  if ratio_hw_b <= 4.0:
      coeff = 0.25
  elif ratio_hw_b >= 6.0:
      coeff = 0.20
  else:
      # 4~6 线性插值
      coeff = 0.25 - 0.05 * ((ratio_hw_b - 4.0) / 2.0)

   **2. 计算截面上限**
  V_max = coeff × fc × b × h0 / γd

  参数说明：
  - coeff：系数（0.20-0.25），高宽比越大系数越小
  - fc：混凝土轴心抗压强度设计值
  - b：截面宽度
  - h0：截面有效高度
  - γd：结构重要性系数

  4. 设计逻辑

  抗剪验算双重检查：

1.    **承载力检查**
*   V_capacity = Vc + Vs  # 混凝土+拉筋抗剪能力
*   check1 = V_capacity >= V_d

2.    **截面上限检查**
*   check2 = V_max >= V_d

3.    **最终判定**
*   is_safe = check1 and check2

  三种情况：

  1. V_d < V_capacity < V_max：正常设计
    - 拉筋配置合理
    - 截面尺寸充足
  2. V_capacity < V_d < V_max：拉筋不足
    - 增加拉筋配置
    - 减小拉筋间距或增加肢数
  3. V_d > V_max：截面超限
    - 拉筋无法解决
    - 必须增大截面尺寸（b或h）

  5. 工程意义

  为什么需要截面限制比：

  1. 防止脆性破坏
    - 斜压破坏是脆性破坏，无预兆
    - 必须通过截面尺寸控制
  2. 指导截面设计
    - limit_ratio > 0.9：截面偏小，建议增大
    - limit_ratio < 0.5：截面富余，可能过于保守
  3. 优化决策依据
    - 拉筋优化时监控此指标
    - 确保所有方案都满足截面限制

  6. 在拉筋优化中的应用

  代码位置：src/structural_solver.py 第434行

  limit_ratio = raw['V_d'] / raw['V_max']
  max_section_limit_ratio = max(max_section_limit_ratio, limit_ratio)

  作用：
  - 记录所有荷载工况中的最大截面限制比
  - 作为拉筋方案的评价指标
  - 输出到帕累托前沿供用户参考

**步骤2：确定控制配箍率**

# 最小配箍率约束

### 2.2 生成候选方案（第457行）

调用 `_build_stirrup_candidates` 从预定义规格体系生成候选：

**规格体系**：
- 钢筋等级：HPB300, HRB400
- 直径：6, 8, 10, 12mm
- 间距：100-300mm（步长10-50mm）
- 肢数：2肢, 4肢

**筛选条件**：
- 配箍率 `Asv/s ≥ required_asv_over_s`
- 满足构造要求

### 2.3 验算并计算目标函数（第460-501行）

对每个候选方案：

**步骤1：抗剪验算**

```python
for forces in force_envelope:
    res = calc.verify_shear(section_params, forces, trial_rebar)
    max_ratio = max(max_ratio, res['ratio'])
    max_limit_ratio = max(max_limit_ratio, res['limit_ratio'])
    if not res['is_safe']:

```

**步骤2：计算拉筋成本**（新增）

```python
stirrup_cost = _calc_stirrup_cost(trial_stirrup, section_params)
```

成本计算公式：
```
拉筋周长 = 2 × (b + h) / 1000  (m)
每米拉筋数量 = 1000 / spacing
每米钢筋重量 = weight_per_m × 周长 × 肢数 × 数量
成本 = (重量 / 1000) × 单价  (元/m)
```

**步骤3：构建目标函数**

```python
objs = (
    stirrup_cost,              # f1: 最小化成本（元/m）
    -trial_stirrup['spacing'], # f2: 最大化间距（mm）
    trial_stirrup['diameter'], # f3: 最小化直径（mm）
)
```

**步骤4：筛选**

仅保留 `all_pass=True` 的方案到候选结果集。

### 2.4 帕累托前沿筛选（第503-522行）

**步骤1：非支配排序**

调用 `_pareto_front` 筛选非支配解：
- 方案A支配方案B：A在所有目标上不差于B，且至少一个目标严格优于B
- 返回所有非支配解

**步骤2：排序**

按 `(cost, -spacing, diameter)` 排序帕累托前沿。

**步骤3：构建输出**

每个帕累托解包含：
```python

    'diameter': 直径,
    'spacing': 间距,
    'grade': 等级,
    'legs': 肢数,
    'provided_Asv_over_s': 配箍率,
    'max_ratio': 最大利用率,
    'cost': 成本,              # 新增
    'spacing_value': 间距值,   # 新增
    'diameter_value': 直径值,  # 新增
}
```

### 2.5 选择默认方案（第524-537行）

从帕累托前沿中选择默认方案，优先级：

1. 配箍率最接近需求值 `|provided - required|` 最小
2. 直径较小
3. 间距较大

```python
chosen = min(pareto_front, key=lambda x: (
    abs(x['provided_Asv_over_s'] - required_asv_over_s),
    x['stirrup']['diameter'],
    -x['stirrup']['spacing'],
))
```

### 2.6 关键改进

**修改前**：目标函数 `(Asv_over_s, -spacing, diameter)`
- 问题：配箍率不能直接反映经济性

**修改后**：目标函数 `(cost, -spacing, diameter)`
- 优势：直接优化经济性，帕累托前沿提供成本-施工性平衡方案

## 三、分布筋配置流程（规则式配置）

**代码位置**：`src/structural_solver.py` 的 `_design_distribution_rebar` 方法

### 3.1 输入参数

- `main_rebar`：主筋配置方案
- `section_params`：截面参数
- `stirrup`：拉筋配置方案（可选）

### 3.2 配置模式

分布筋支持两种配置模式：

#### 模式1：固定规格模式（第266-297行）

**触发条件**：用户指定 `dist_diameter` 和 `dist_count`

**流程**：
1. 使用指定的直径和根数
2. 根据截面宽度反算间距
3. 校核构造要求：
   - 间距 ≤ 最大允许间距
   - 净距 ≥ 最小净距

**输出**：
```python
{
    'mode': 'fixed',
    'diameter': 指定直径,
    'count': 指定根数,
    'spacing': 反算间距,
    'is_construct_ok': 构造检查结果,
}
```

#### 模式2：比例配置模式（第299-355行）

**触发条件**：未指定固定规格

**流程**：

**步骤1：计算所需面积**

```python
ratio = section_params.get('dist_ratio_to_main', 0.15)
as_required = ratio × as_main
```

默认分布筋面积为主筋面积的15%。

**步骤2：确定搜索范围**

```python
# 直径范围
min_dist_dia = 8mm（默认）
max_dist_dia = 25mm（默认）
diameters = [10, 12, 14, 16, 18, 20, 22, 25]

# 最大间距
max_dist_spacing = 300mm（默认）
```

**步骤3：可选-跟随拉筋间距**

```python
if dist_follow_stirrup and stirrup:
    max_dist_spacing = min(max_dist_spacing, stirrup['spacing'])
```

**步骤4：遍历直径候选**（第318-352行）

对每个直径：

1. 计算有效宽度：`b_eff = b - 2×cover - diameter`

2. 计算所需根数：
   ```python
   # 按间距要求
   min_count_by_spacing = ceil(b_eff / max_dist_spacing) + 1
   
   # 按面积要求
   min_count_by_area = ceil(as_required / area)
   
   # 取较大值
   count = max(min_count_by_spacing, min_count_by_area, 2)
   ```

3. 计算实际间距：
   ```python
   actual_spacing = b_eff / (count - 1)
   ```

4. 构造检查：
   - 净距 ≥ 最小净距
   - 提供面积 ≥ 所需面积

5. 计算评价指标：
   ```python
   overuse = as_provided - as_required
   key = (overuse, diameter, -actual_spacing)
   ```

**步骤5：选择最优方案**

选择评价指标最小的方案：
- 优先：超配最少
- 其次：直径较小
- 最后：间距较大

### 3.3 降级方案（第357-378行）

**触发条件**：无满足要求的方案

**策略**：
- 使用最小直径
- 采用密集间距（80-100mm）
- 尽可能提供足够的分布筋面积

**标记**：

## 四、三阶段配筋流程对比

| 特征 | 主筋 | 拉筋 | 分布筋 |
|-----|------|------|--------|
| **算法类型** | NSGA-II多目标优化 | 启发式+帕累托筛选 | 规则式配置 |
| **搜索空间** | 288种组合 | 数百种组合 | 数十种组合 |
| **计算复杂度** | 高（200×200代） | 中（遍历候选） | 低（直接计算） |
| **优化目标** | 成本、裂缝、间距 | 成本、间距、直径 | 超配最少 |
| **输出形式** | 帕累托前沿 | 帕累托前沿 | 单一方案 |
| **验算内容** | 承载力+裂缝 | 抗剪 | 构造要求 |
| **归一化** | 是（Min-Max） | 否 | 不适用 |

## 五、关键技术点

### 5.1 目标函数归一化

**问题**：不同目标量纲差异大
- 成本：10-100元/m
- 裂缝：0-0.2mm
- 间距：100-250mm

**解决方案**：Min-Max归一化到[0,1]

```python
normalized[i] = (objs[i] - min[i]) / (max[i] - min[i])
```

**效果**：防止成本主导优化，确保多目标平衡。

### 5.2 帕累托前沿筛选

**定义**：非支配解集合
- 方案A支配方案B：A在所有目标上不差于B，且至少一个目标严格优于B
- 帕累托前沿：所有非支配解

**应用**：
- 主筋：提供成本-裂缝-间距的平衡方案
- 拉筋：提供成本-间距-直径的平衡方案

### 5.3 种群规模配置

**当前配置**：200×200（重要项目）

**理由**：
- 搜索空间288种组合
- 200个体覆盖69%空间
- 计算时间约67分钟
- 帕累托前沿质量高

### 5.4 成本计算

**主筋成本**：
```
成本 = (weight_per_m × count / 1000) × price_unit
```

**拉筋成本**：
```
周长 = 2 × (b + h) / 1000
数量 = 1000 / spacing
重量 = weight_per_m × 周长 × 肢数 × 数量
成本 = (重量 / 1000) × 单价
```
